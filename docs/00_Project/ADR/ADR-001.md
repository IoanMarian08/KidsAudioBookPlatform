# ADR-001: Use Flutter for Mobile Development

| Field | Value |
|---|---|
| Status | Accepted |
| Date | 2026-07-04 |
| Owner | Ioan Marghioala |
| Related Area | Foundation Architecture |

---

## Context

The product must be available on both Android and iOS. The team is small and needs a single codebase to reduce development time while maintaining a high-quality mobile experience.

---

## Decision

Use Flutter as the main mobile framework for Android and iOS.

---

## Alternatives Considered

- Native Android + Native iOS
- React Native
- Flutter

---

## Consequences

- Single codebase
- Faster development
- Consistent UI
- Requires Flutter expertise
- iOS build still requires macOS/Xcode

---

## Notes

This decision may be revisited if product scale, team size or business constraints change.
