# ADR-002: Use Spring Boot Microservices for Backend

| Field | Value |
|---|---|
| Status | Accepted |
| Date | 2026-07-04 |
| Owner | Ioan Marghioala |
| Related Area | Foundation Architecture |

---

## Context

The platform needs authentication, content management, media, subscriptions, ads, notifications and analytics. These areas may scale independently over time.

---

## Decision

Use Spring Boot for backend services and organize the backend as disciplined microservices.

---

## Alternatives Considered

- Single monolith
- Modular monolith
- Spring Boot microservices
- Node.js services

---

## Consequences

- Clear service boundaries
- Scalable architecture
- More operational complexity
- Requires Docker and service communication standards

---

## Notes

This decision may be revisited if product scale, team size or business constraints change.
