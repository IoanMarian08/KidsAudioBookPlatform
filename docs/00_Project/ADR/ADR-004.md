# ADR-004: Separate Parent Zone from Child Experience

| Field | Value |
|---|---|
| Status | Accepted |
| Date | 2026-07-04 |
| Owner | Ioan Marghioala |
| Related Area | Foundation Architecture |

---

## Context

The product serves children but is controlled by adults. Children must not access subscriptions, settings or sensitive account operations.

---

## Decision

Design Parent Zone as a protected experience separate from Child Experience, accessed through PIN + biometrics.

---

## Alternatives Considered

- Single mixed interface
- Hidden settings menu
- Separate protected Parent Zone

---

## Consequences

- Improved child safety
- Clear UX boundary
- More implementation work
- Supports future parental controls

---

## Notes

This decision may be revisited if product scale, team size or business constraints change.
